## 1. Introduction

This document defines the logging and record-keeping requirements for ADS-DV operation.

It is intended to be used alongside the current User Manual, Operational Safety, RAMS, and the [FS-AI ADS-DV Glossary](./FS-AI%20ADS-DV%20Glossary.md). This document does not replace the operating and safety procedures in those documents. It defines how run records are to be captured and how the GEMS datalogger data is to be managed.

## 3. Logging Scope

The ADS-DV Operator must ensure that appropriate run logging is completed whenever ADS-DV operation is undertaken.

In particular:

- A run logging record sheet must be completed whenever a logged ADS-DV run or operating session is carried out.
- Dynamic Operation logging using the GEMS DA3 Datalogger must be supervised by the ASR, in accordance with [FS-AI ADS-DV Operational Safety, Section 6](./FS-AI%20ADS-DV%20Operational%20Safety.md#6-dynamic-operation).
- The logging record and the downloaded GEMS DA3 Datalogger data together form the operational record of the run or session.


## 4. Run Logging Record Sheet

A Run Logging Record Sheet should be completed clearly and retained as part of the operation record for the ADS-DV Operator.

Where multiple runs are carried out in one session, the record must still identify each run clearly and any relevant anomaly, interruption, or abnormal outcome must be recorded.

> **Note:** The run logging record sheet will be provided with the ADS-DV or made available online. It must be completed by the ASR whenever the ADS-DV is operated.


## 5. GEMS Datalogger Data Management

The ADS-DV uses a GEMS DA3 Datalogger with a Memory Card to record run data during operation.

The GEMS DA3 Datalogger memory must be managed so that sufficient storage capacity remains available for continued operation.

The following controls apply:

- Use the current [FS-AI ADS-DV User Manual, Section 6](./FS-AI%20ADS-DV%20User%20Manual.md#6-autonomous-operation) for the correct insertion and removal of the GEMS DA3 Datalogger Memory Card.
- Remove the GEMS DA3 Datalogger Memory Card from the ADS-DV as required and download the recorded data to a Windows PC.
- Use the GEMS GDA software on that Windows PC to access and download the logged data from the GEMS DA3 Datalogger Memory Card.
- During periods of repeated running, download data at suitable intervals so that adequate free storage space remains available on the GEMS DA3 Datalogger Memory Card.
- Do not continue Dynamic Operation if GEMS DA3 Datalogger storage management has not been maintained and sufficient free storage space cannot be confirmed.

> **Note:** The GEMS DA3 Datalogger data should be downloaded successfully before the GEMS DA3 Datalogger Memory Card is returned to service for further ADS-DV operation.


## 6. Record Keeping

The ADS-DV Operator should retain the completed run logging record sheet together with the downloaded GEMS DA3 Datalogger data as part of the operation record.

Where additional documents are required for the activity, venue, or organisation, these records should be retained alongside the relevant RAMS and other supporting safety documentation.
